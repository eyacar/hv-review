Formatting inconsistency
1
Two-row header — row 1 has doc name + actions (submit, upload new version), row 2 has step indicator + metadata. Keeps the primary action visible without crowding the title.
2
Step indicator shows Upload → Processing → Review → Submitted. Gives users orientation — they know where they are and what comes next. Derived from the status field, not hard-coded.
3
Toolbar above the scroll area — zoom + pagination controls sit between the header and the PDF, never floating over content. Avoids z-index conflicts and keeps the reading area unobstructed.
4
Issues panel fixed width (380px) — wide enough to read descriptions without wrapping, narrow enough to leave substantial space for the document. Filter chips reduce need to scroll the list.
5
Active page ring — the current page gets a blue outline. Clicking an issue navigates the PDF to that page and highlights it, creating a direct connection between the issue list and the document.

A
Tab bar at bottom — not a top tab pattern. iOS and Android both use bottom navigation for primary switching. The badge on Issues communicates blocking count without the user needing to switch tabs.
B
Auto-switch on issue tap — tapping an issue card switches to Document tab AND scrolls to that page. State lives in Zustand so IssueCard can trigger it without prop drilling through IssuesPanel.
C
Step indicator — mobile — all step labels are hidden except the current step. Four dots with one visible label gives orientation without wrapping. Connectors shrink from 24px to 12px.

Key design decisions

Decision Chosen approach Alternatives considered Reason
PDF rendering react-pdf + text layer <iframe>, pdf.js directly Text layer = native CMD+F. iframe gives up all programmatic control (page scroll, active highlight).
PDF text search Browser native CMD+F Custom search UI Native search is instant, keyboard-accessible, and users already know it. Custom search adds complexity for no gain.
Shared state Zustand store React Context, prop drilling Viewer and issues panel are siblings. Zustand avoids lifting state and threading props through intermediaries.
Mobile layout Tab bar (bottom) Drawer, split at 50/50, top tabs On 375px width neither panel has enough space side-by-side. Bottom tabs follow native mobile conventions.
Submission blocking aria-disabled + aria-describedby HTML disabled attribute disabled removes the button from keyboard navigation. aria-disabled keeps it focusable so screen readers can read why it's blocked.
Issue severity color Color + icon + text label Color only WCAG 1.4.1: information cannot be conveyed by color alone. Icon + text label ensures it works for color-blind users.
Horizontal PDF scroll at zoom align-items: flex-start + margin: auto align-items: center Center alignment overflows both sides — browser scroll origin is 0 so left overflow is unreachable. flex-start constrains overflow to the right only.
